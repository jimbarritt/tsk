# Session Log — 2026-03-09

## TUI — Box-drawing table

Replaced the ratatui `Table` widget approach (which produced misaligned output) with a
manually-constructed `Paragraph` of `Line` objects using Unicode box-drawing characters
(`┌┬┐`, `├┼┤`, `└┴┘`). All borders and separators are now rendered consistently.

## TUI — Section-based layout

Threads are grouped into three sections rendered as separate tables, each with its own
title and box:

- `> Active`
- `> Priority & Incidents`
- `> Background`

Empty sections are omitted entirely. A blank line separates sections.

## TUI — Responsive column layout

Columns are ordered by importance: `ID │ SLUG │ STATE │ PRIO │ DESC`.

Slug is always rendered at full width (`SLUG_FULL = 40`) when any optional column is
visible. Optional columns appear in order as space allows and drop in reverse order as
the terminal narrows:

1. Wide: all five columns, description fills surplus
2. Slug + STATE + PRIO
3. Slug + STATE
4. Slug only (expands to fill)

## TUI — Bug fixes

**BUG-1: Unicode panic on resize.** `pad()` was slicing strings by byte index, which
panicked when a multi-byte character (e.g. em dash `—`) fell on a column boundary.
Fixed by using `chars().count()` and `chars().take(w).collect()`.

**BUG-1: Terminal not restored on panic.** Installed a panic hook before `enable_raw_mode()`
that calls `disable_raw_mode()` and `LeaveAlternateScreen`, ensuring the terminal is always
restored even if the event loop panics.

## TUI — State refresh (ADR 0005 → 0006)

Initial implementation watched `index.json` via the `notify` crate (kqueue/inotify). This
proved unreliable: kqueue directory watches fire on structural changes (file
creation/deletion) but not on content modifications to existing files. `switch-to` updates
`index.json` in place, so the watch never fired.

Replaced with simple 500ms polling via `thread.list` over the daemon socket. The `notify`
crate was removed. ADR 0006 documents this decision and flags daemon push (ADR to be written)
as the correct long-term solution.

## `thread start` → `thread create`

Renamed `tsk thread start` to `tsk thread create`. New threads are created in `Paused`
state and do not change the currently active thread. Use `tsk thread switch-to` to activate.

Rationale: "start" implied the thread would become active immediately, which conflated
creation with activation.

## Numeric thread IDs (hash removed)

Replaced `hash` / `short_hash` (SHA-256 derived) with a stable monotonic `id: u32`,
zero-padded to 4 digits (`0001`, `0002`, ...).

- Thread directories: `tsk/threads/0001-fix-login/`
- `switch-to` resolves by id (`1`, `0001`) or slug
- TUI displays id as a separate column
- Hash-based identification removed from all user-facing surfaces
- SHA-256 is still used internally for the daemon socket path

## `index.md` scaffolding

`tsk thread create` now writes an `index.md` into the new thread directory, pre-populated
with id, slug, priority, creation date, and description. Provides an immediate place to
start capturing context without any manual setup.

## `tsk context` command

Added `tsk context` — a top-level command that prints a mini system prompt for agents,
followed by the live thread list from the daemon (or a note if the daemon is not running).

The prompt source is `cli/src/agent-context.md`, embedded at compile time via
`include_str!()`. Edit the markdown file and rebuild to update the prompt — no Rust changes
needed.

The prompt covers:
- Core concepts (threads, directories, priorities, commands)
- Response formats for `thread.create`, `thread.list`, `thread.switch-to`
- **Mode A** — autonomous agent: agent owns its own threads and context
- **Mode B** — human-assisted: agent is co-pilot, human leads

## Workspace version management

Moved `version`, `homepage`, `repository`, `license`, `keywords`, `categories`, and
`edition` into `[workspace.package]` in the root `Cargo.toml`. All three crates inherit
via `field.workspace = true`. The `tsk-core` inter-crate dependency is declared once in
`[workspace.dependencies]`.

To bump the version, change two lines in the root `Cargo.toml`.

## `just publish` recipe

Added a `publish` recipe to the justfile that publishes all three crates in dependency
order (`tsk-core` first, 30-second sleep, then `tsk-bin` and `tskd`). The sleep is
required for crates.io to index `tsk-core` before the dependents resolve the version.
